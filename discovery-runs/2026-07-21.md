# Discovery run 2026-07-21

## Scope
Daily autonomous askhermie.dev discovery loop. Inspected GitHub/web candidates across observability, AI-agent tooling, MCP/local AI, ESP32 hardware learning, homelab dashboards, and Docker maintenance.

## Pre-existing working tree
The run started with uncommitted site work already present: Svelte island integration, `ResourceExplorer.svelte`, package updates, README note, and two pending Svelte/SvelteKit resource cards. I inspected the diff, kept it because it is safe and directly related to the resource catalog UX, and added missing safety notes before committing.

## Accepted / published

1. **Svelte** — already pending in working tree; accepted after adding a safety note. Benign UI framework; official install path; useful for Astro islands and copy/search UI.
2. **SvelteKit** — already pending in working tree; accepted after adding a safety note. Benign web framework; included with a conservative “do not migrate by default” first move.
3. **SigNoz** — accepted. OpenTelemetry observability platform; active repo, strong adoption, self-host docs. Safety note warns that telemetry can expose logs, URLs, and headers.
4. **Vector** — accepted. Observability data pipeline; active repo, documented quickstart. Safety note warns about log redaction and third-party sinks.
5. **Jaeger** — accepted. CNCF distributed tracing platform; README provides a safe all-in-one Docker quickstart. Safety note warns traces may expose service details.
6. **CodeBurn** — accepted. Local-first AI coding token/cost tracker; README says it reads local session files and does not require API keys. Safety note warns session files can contain private prompts/code.
7. **MCP Client for Ollama** — accepted. Local MCP client for Ollama; useful for safe local MCP testing. Safety note warns that MCP tools can touch files/services and should be allowlisted.
8. **ESP32 MPY-Jama** — accepted. Cross-platform ESP32 MicroPython IDE/file manager/REPL. Safety note limits use to owned boards and disconnected loads while testing.
9. **ESP32 CSI Tool** — accepted. Research/learning tool for ESP32 Wi-Fi CSI collection. Safety note limits to owned boards/networks and consented experiments.
10. **Dashy** — accepted. Self-hosted homelab dashboard; active repo and simple Docker run. Safety note warns not to expose admin dashboards publicly.
11. **Tugtainer** — accepted. Docker update-check UI; README explicitly says automatic updates are disabled by default and not recommended for production. Safety note warns Docker socket access is powerful and unattended updates are risky.

## Inspected but already present

## Rejected / deferred

1. **ESP32 Marauder** (`justcallmekoko/ESP32Marauder`) — rejected. README markets it as Wi-Fi/Bluetooth offensive tooling; too easy for beginners to misuse.
2. **nRFBox** (`cifertech/nRFBox`) — rejected. README/wiki mention jamming, spoofing, deauthentication, and signal replay; offensive/illegal-use risk is too high.
3. **ESP32-Bit-Pirate** (`geo-tp/ESP32-Bit-Pirate`) — deferred, not published. Hardware protocol tool may be legitimate, but I did not complete enough safety review in this run.
4. **Tugtainer production auto-update usage** — accepted only with warnings; unattended production update workflow is deferred as unsafe for beginners.
